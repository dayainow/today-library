## 전체 설명 (4,000자 이내)

```
오늘의 도서관은 전국 공공도서관 3,400여 곳의 실시간 운영 정보를 내 위치 기반으로
거리순·열림순으로 보여주는 도서관 탐색 앱입니다.

▶ 주요 기능

• 지금 열린 도서관 우선 정렬 — 현재 시간 기준으로 문 열린 곳을 맨 위에 표시
• 내 위치 기반 거리 계산 — GPS를 허용하면 걸어서 몇 km인지 바로 확인
• 빠른 검색 — 지역명(마포, 강남), 도서관 이름, 종류(어린이, 작은)로 즉시 검색
• 필터 탭 — 전체 / 지금 열림 / 주말 운영 / 즐겨찾기 4가지 탭으로 필터
• 즐겨찾기 — 자주 가는 도서관을 별도로 저장하고 빠르게 접근
• 운영시간 요약 — 평일·토·일 운영시간과 좌석 수를 카드 한 장에 표시
• 전화·길찾기·홈페이지 — 바로 전화 연결, 지도 앱 열기, 공식 홈페이지 이동 지원

▶ 데이터

공공데이터포털(data.go.kr) 공공도서관 API를 매일 자동 갱신합니다.
네트워크가 없을 때도 내장 데이터로 기본 정보를 제공합니다.

▶ 권한 안내

• 위치(선택) — 가까운 도서관을 거리순으로 정렬할 때만 사용합니다.
  허용하지 않아도 앱의 모든 기능을 이용할 수 있습니다.
```

## 스크린샷 체크리스트

Play Store는 폰 스크린샷 최소 2장, 최대 8장을 요구합니다. (권장 해상도: 1080×1920 또는 1080×2340)

| # | 화면 | 촬영 시 보여줄 내용 |
|---|---|---|
| 1 | 홈 (전체) | 히어로 타이틀 + 열린 도서관 개수 + 카드 목록 |
| 2 | 홈 (지금 열림 탭) | 필터 칩 활성 상태 + 열린 도서관만 표시 |
| 3 | 홈 (검색) | 검색창에 "마포" 입력, 필터된 결과 |
| 4 | 도서관 카드 상세 | 운영시간 그리드 + 전화/길찾기 버튼 |
| 5 | 즐겨찾기 탭 | 저장된 도서관 목록 |
| 6 | 위치 권한 안내 | 서울시청 기준 → 현재 위치 전환 안내 |

### 스크린샷 촬영 방법

```bash
# Android 에뮬레이터에서 스크린샷 저장
adb exec-out screencap -p > screenshot_01.png
```

또는 Android 실기기에서 볼륨 다운 + 전원 버튼으로 캡처 후 USB로 꺼냄.

### Feature Graphic 제작 가이드

Play Store 앱 상단에 표시되는 1024×500 배너입니다. Figma, Canva, Adobe Express 등에서 제작합니다.

**권장 레이아웃**

```
┌────────────────────────────────────────────────────┐  500px
│  (배경: #f7f8fb 또는 #2563eb 그라데이션)            │
│                                                    │
│   ┌──┐  오늘의 도서관          [파란 브랜드 블록] │
│   │🏛 │  전국 도서관 3,400곳   작은 카드 미리보기 │
│   └──┘  지금 열려 있는 곳만    (우측 배치)        │
│                                                    │
└────────────────────────────────────────────────────┘
                     1024px
```

**색상 팔레트**

| 용도 | 색상 |
|---|---|
| 배경 (밝은 버전) | `#f7f8fb` |
| 배경 (진한 버전) | `#0b1220` |
| 브랜드 파란색 | `#2563eb` |
| 열림 초록색 | `#059669` |
| 텍스트 (밝은 배경 위) | `#0b1220` |
| 텍스트 (어두운 배경 위) | `#ffffff` |

**Canva 빠른 제작 방법**

1. Canva → 새 디자인 → 사용자 지정 크기 → 1024 × 500
2. 배경 `#f7f8fb` 설정
3. 왼쪽: 파란 사각형(`#2563eb`, 32×32, 둥근 모서리 8) + 텍스트 "오늘의 도서관" (굵게, 32px)
4. 왼쪽 아래: "전국 공공도서관 3,400곳 · 지금 열린 곳 바로 찾기" (회색, 18px)
5. 오른쪽: 앱 스크린샷 이미지 삽입 (각도 약간 기울임)
6. PNG로 내보내기

## 개인정보 처리방침 요구사항

Play Store는 위치 권한을 사용하는 앱에 개인정보 처리방침 URL을 **필수**로 요구합니다.

### 최소 포함 내용

```
1. 수집하는 정보: 위치 정보 (선택적)
2. 수집 목적: 가까운 도서관 거리 계산
3. 제3자 제공: 없음
4. 보관 기간: 앱 실행 중에만 사용, 서버에 저장하지 않음
5. 문의: dayainow@gmail.com
```

### 배포 URL (구현 완료)

`api/privacy.ts`로 Vercel 라우트 구현 완료. 배포 후 아래 URL이 활성화됩니다.

```
https://today-library-sigma.vercel.app/api/privacy
```

Play Console → 앱 콘텐츠 → 개인정보 처리방침에 위 URL을 입력하세요.

> 커스텀 도메인을 사용하는 경우 도메인에 맞게 URL을 수정하세요.

## EAS 빌드 명령어 (Android 출시)

```bash
# 1. EAS CLI 설치 (최초 1회)
npm install -g eas-cli

# 2. EAS 로그인
eas login

# 3. production 빌드 (AAB 생성, Google Play 제출용)
eas build --platform android --profile production

# 4. (선택) Google Play에 직접 제출
eas submit --platform android --latest
```

> `eas.json`의 `production.autoIncrement: true`로 빌드마다 versionCode가 자동 증가합니다.
